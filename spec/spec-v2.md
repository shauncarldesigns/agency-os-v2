## 1. What This Is

A web app for running a small web design agency that targets local service businesses. The app is one person's complete operations layer:

- **Prospects** scored automatically from Google Places data
- **Pipeline** for cold-call CRM with full call history per lead
- **Briefs** generated by Claude API and handed to **Cowork** to drive landingsite.ai
- **Sites** managed via tier-aware cards with SEO coverage matrix
- **Reports** auto-generated monthly for Tier 3 clients (Search Console + PageSpeed + Cloudflare)

The app does NOT build websites itself. **Cowork drives landingsite.ai's UI to build them**, while the app orchestrates the workflow, briefs, and reporting.

## 2. The Three-Tier Pricing Model

The entire product is built around three tiers that determine workflow:

| | Tier 1 | Tier 2 | Tier 3 |
|---|---|---|---|
| Build cost | $800 one-time | $400 one-time | Free |
| Monthly | $0 | $79 | $499 |
| Contract | None | Cancel anytime | 6-month minimum |
| Pages at launch | 5 | 5 | 8-10 + foundation |
| Monthly site work | None — handoff complete | Hosting + edit requests handled by Shaun | 3 service-area SEO pages from Shaun |
| External services | None | None | Merchynt: GBP rep + listing management |
| Reporting | None | None | Monthly Search Console + PageSpeed report |

**Critical:** Merchynt is white-labeled. The word "Merchynt" must NEVER appear in client-facing content (emails, PDFs, dashboards). Internal data tracks Merchynt; external content describes work generically ("GBP management," "review responses").

## 3. Architecture Overview

```
┌─ Cloudflare Workers (backend API)
│  ├── D1 database — leads, projects, sites, briefs, page queue, snapshots
│  ├── Cloudflare Queues — bridge between cloud API and local Cowork
│  ├── Claude API — brief generation, copy, monthly summaries, review mining
│  ├── Google Places API — prospect search + GBP details + reviews
│  ├── PageSpeed Insights API — website quality scoring
│  ├── Search Console API (OAuth) — Tier 3 reporting
│  ├── Cloudflare Analytics API — traffic data for Tier 3 reports
│  └── Cron triggers — monthly Tier 3 page generation, snapshot creation
│
├─ React + Vite dashboard on Cloudflare Pages
│  └── Five tabs: Prospect, Pipeline, Build, Sites, Reports
│
└─ Cowork worker (runs on local desktop or always-on Mac mini)
   ├── Polls Cloudflare Queues for jobs
   ├── Drives landingsite.ai UI (open project, paste brief, wait for build)
   └── Reports completion back to API via webhook
```

**What's NOT in v2 (removed from v1):**
- Astro template + Keystatic CMS
- GitHub repo creation/management for client sites  
- R2 storage for site source files
- WebP conversion pipeline
- Image generation (Nano Banana, Unsplash, Pexels) — landingsite.ai handles imagery
- Cloudflare Pages deployment for client sites — landingsite.ai hosts everything
- Build orchestration for Astro → dist → Cloudflare upload

### 5.5 Opportunity Scoring Algorithm

`services/scoring.ts` — calculate 0-100 score from these signals:

```typescript
interface ScoringInput {
  hasWebsite: boolean;
  pagespeedMobile: number | null;        // 0-100
  pagespeedDesktop: number | null;
  gbpClaimed: boolean;
  gbpPhotos: number;
  gbpHasDescription: boolean;
  gbpHasHours: boolean;
  reviewCount: number;
  rating: number | null;
  recentReviewActivity: boolean;          // any review in last 90 days
  yearsInBusiness: number | null;
}

function calculateOpportunityScore(input: ScoringInput): {
  score: number;
  tier: 1 | 2 | 3;
  reasoning: string;
} {
  let score = 0;
  const factors: string[] = [];

  // Website signals (weight: 30 points)
  if (!input.hasWebsite) {
    score += 25;
    factors.push("No website — easy Tier 1 pitch");
  } else if (input.pagespeedMobile !== null && input.pagespeedMobile < 50) {
    score += 25;
    factors.push(`Slow website (PSI ${input.pagespeedMobile}) — clear pain point`);
  } else if (input.pagespeedMobile !== null && input.pagespeedMobile < 70) {
    score += 15;
    factors.push("Decent but improvable site");
  }

  // GBP signals (weight: 30 points)
  if (!input.gbpClaimed) {
    score += 25;
    factors.push("Unclaimed GBP — high opportunity");
  } else {
    if (input.gbpPhotos < 5) score += 10;
    if (!input.gbpHasDescription) score += 5;
    if (!input.gbpHasHours) score += 5;
  }

  // Review/authority signals (weight: 30 points)
  if (input.reviewCount >= 50) {
    score += 25;
    factors.push(`${input.reviewCount} reviews — established business`);
  } else if (input.reviewCount >= 20) {
    score += 18;
  } else if (input.reviewCount >= 5) {
    score += 10;
  }
  if (input.rating !== null && input.rating >= 4.5) score += 5;
  if (input.recentReviewActivity) score += 5;

  // Years in business (weight: 10 points)
  if (input.yearsInBusiness && input.yearsInBusiness >= 10) score += 10;
  else if (input.yearsInBusiness && input.yearsInBusiness >= 5) score += 5;

  score = Math.min(100, score);

  // Tier recommendation
  let tier: 1 | 2 | 3;
  if (score >= 75 && input.reviewCount >= 30) {
    tier = 3;  // Established business, real revenue, ready to invest in growth
  } else if (score >= 50 || (input.hasWebsite && input.pagespeedMobile && input.pagespeedMobile < 60)) {
    tier = 2;  // Has some presence, needs ongoing help
  } else {
    tier = 1;  // Quick foundation, transactional
  }

  return { score, tier, reasoning: factors.join(" · ") };
}
```

### 5.8 Cowork Worker (`/cowork-worker/`)

Separate Node.js script that runs on the user's local machine.

**Functionality:**

```typescript
// cowork-worker/src/index.ts
async function pollQueue() {
  while (true) {
    // Pull next job from Cloudflare Queue
    const job = await fetchNextJob();
    if (!job) {
      await sleep(30_000);  // 30s between polls when idle
      continue;
    }

    // Notify backend job started
    await notifyStarted(job.id);

    try {
      // Drive Cowork desktop agent to:
      // 1. Open landingsite.ai
      // 2. Log in if needed
      // 3. Navigate to project (or create new project)
      // 4. Send the brief as chat prompt
      // 5. Wait for "page complete" indicator
      // 6. Handle "continue?" prompts as needed
      // 7. Return page URL once built
      const result = await coworkExecute(job);

      await notifyCompleted(job.id, { success: true, pageUrl: result.url });
    } catch (err) {
      await notifyCompleted(job.id, { success: false, error: err.message });
    }
  }
}
```

**Important — Cowork integration approach:**

For v2 launch, the Cowork integration may be **manual-trigger** rather than fully automated. The workflow:

1. App generates the brief and queues it
2. App displays the brief and a "Hand off to Cowork" button
3. User clicks the button which copies the brief to clipboard
4. User opens Cowork themselves and pastes the brief
5. User watches Cowork build the page
6. User clicks "Mark complete" in the app when done

This manual trigger is significantly easier to build than full polling automation, and lets Shaun validate the workflow before automating. **Build the manual-trigger version first**. Add full automation in a follow-up phase once the workflow is proven.

```typescript
// src/lib/api.ts
const API_BASE = import.meta.env.VITE_API_URL;
const API_KEY = import.meta.env.VITE_API_KEY;

async function apiFetch<T>(path: string, options: RequestInit = {}): Promise<T> {
  const res = await fetch(`${API_BASE}${path}`, {
    ...options,
    headers: {
      'Content-Type': 'application/json',
      'X-API-Key': API_KEY,
      ...options.headers,
    },
  });
  if (!res.ok) {
    const err = await res.json().catch(() => ({ error: res.statusText }));
    throw new ApiError(err.error, res.status, err);
  }
  return res.json();
}

export const api = {
  leads: {
    list: (filters?) => apiFetch<Lead[]>(`/api/leads?${qs(filters)}`),
    get: (id) => apiFetch<LeadWithCalls>(`/api/leads/${id}`),
    create: (data) => apiFetch<Lead>('/api/leads', { method: 'POST', body: JSON.stringify(data) }),
    update: (id, data) => apiFetch<Lead>(`/api/leads/${id}`, { method: 'PUT', body: JSON.stringify(data) }),
    delete: (id) => apiFetch(`/api/leads/${id}`, { method: 'DELETE' }),
    importCsv: (formData) => fetch(`${API_BASE}/api/leads/import`, { method: 'POST', body: formData, headers: { 'X-API-Key': API_KEY } }),
    enrich: (id) => apiFetch(`/api/leads/${id}/enrich`, { method: 'POST' }),
    enrichAll: () => apiFetch('/api/leads/enrich-all', { method: 'POST' }),
  },
  calls: {
    list: (leadId) => apiFetch<CallEntry[]>(`/api/leads/${leadId}/calls`),
    create: (leadId, data) => apiFetch<CallEntry>(`/api/leads/${leadId}/calls`, { method: 'POST', body: JSON.stringify(data) }),
  },
  prospect: {
    search: (input) => apiFetch<ProspectResults>('/api/prospect/search', { method: 'POST', body: JSON.stringify(input) }),
    addToPipeline: (placeIds) => apiFetch('/api/prospect/add-to-pipeline', { method: 'POST', body: JSON.stringify({ placeIds }) }),
  },
  projects: {
    list: () => apiFetch<Project[]>('/api/projects'),
    get: (id) => apiFetch<ProjectDetail>(`/api/projects/${id}`),
    create: (data) => apiFetch<Project>('/api/projects', { method: 'POST', body: JSON.stringify(data) }),
    coverage: (id) => apiFetch<CoverageMatrix>(`/api/projects/${id}/coverage`),
  },
  briefs: {
    generate: (input) => apiFetch<{ markdown: string }>('/api/briefs/generate', { method: 'POST', body: JSON.stringify(input) }),

  },
  reports: {
```

## 8. Critical Behavioral Rules

### White-label Protection
- The word "Merchynt" must NEVER appear in:
  - Client-facing PDFs
  - Client-facing emails
  - Any UI element a client could see
- Internal data tracks Merchynt status; external content describes work generically ("GBP management," "review responses")
- Claude prompts for client-facing content (exec summaries, recommendations) must include this rule explicitly

### Pipeline Data → Brief Continuity
The data flow from Prospect → Pipeline → Brief must preserve everything:
- Business name, phone, address
- Review-mined services and service areas (with frequency counts)
- Pitch quotes (best ones flagged)
- Owner names extracted from reviews
- Recommended tier based on scoring

This means the project record (created when lead becomes client) inherits all relevant fields from the lead record — not re-fetched from APIs.

### Cost Discipline
- 5 reviews per prospect from free Places API tier
- For Tier 3 clients post-signing, optionally pull full review history via paid service (Outscraper ~$1-2 per business)
- Don't over-fetch — cache GBP details for 24 hours, PageSpeed for 24 hours
- Claude calls: enrichment is ~$0.005 per lead, brief generation ~$0.10 per site

### Tier 3 Investment
- Tier 3 launch builds get more attention than Tier 2 (they're free + 6-month commitment)
- 8-10 page initial build with full SEO foundation
- Service-area pages in initial build (3-5 of them)
- Brief Claude prompt should specify "this is a Tier 3 launch — invest in SEO depth"
